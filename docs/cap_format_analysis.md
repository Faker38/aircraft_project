# CAP 格式验证记录

## 1. 目的

本记录用于整理当前 `CAP` 样本文件在联调阶段的可用结论，服务于：

- 预处理页头信息预览
- CAP 版预处理算法接入
- 后续样本切片与结果展示

本轮重点是统一联调口径，不把当前结论提前固化为永久标准。

## 2. 分析样本

当前工作区样本如下：

- `IQ_2025_01_09_13_55_30.cap`，大小 `256,000,512` 字节
- `head.cap`，大小 `1,048,576` 字节

已确认：

- `head.cap` 是完整文件前 `1 MB` 截取
- `head.cap` 中仍然完整包含 CAP 头和起始 IQ 数据

## 3. 当前试跑口径

当前联调统一采用以下解释：

```text
┌───────────────────────────────────────┐
│ 文件头（当前按 0x200 = 512 字节试跑） │
├───────────────────────────────────────┤
│ IQ 数据区（从 0x200 到文件末尾）      │
│ 大端 int16，I/Q 交织：I0,Q0,I1,Q1...  │
└───────────────────────────────────────┘
```

说明：

- `0x200` 是当前更优先的联调假设
- 其依据来自：
  - `0x200` 后数据形态更像连续小整数 IQ
  - `(文件大小 - 512) / 4 / 12.8 MHz` 恰好对应 `5 秒`
- 该结论当前用于试跑，不视为永久铁结论

## 4. 当前可稳定读取的头字段

| 偏移 | 长度 | 类型 | 当前值 | 当前解释 |
| --- | --- | --- | --- | --- |
| `0x0000` | 8 字节 | ASCII 字符串（null 结尾） | `B.01.00` | 文件格式版本号 |
| `0x0010` | 8 字节 | `float64` 大端序 | `10000000.0` | 分析带宽 / Span，单位 Hz |
| `0x0018` | 8 字节 | `float64` 大端序 | `1090000000.0` | 中心频率，单位 Hz |
| `0x0110` | 4 字节 | `uint32` 大端序 | `62500` | 疑似每帧采样数 |
| `0x0114` | 4 字节 | `uint32` 大端序 | `1024` | 疑似块大小或 FFT 长度 |

补充说明：

- 导师当前解释为：
  - `10 MHz` 对应可分析信号带宽
  - 实际 IQ 采样率为 `12.8 MHz`
- 当前算法联调按 `sample_rate_hz = bandwidth_hz * 1.28` 执行

## 5. IQ 数据区解释

| 项目 | 当前结论 |
| --- | --- |
| IQ 起始偏移 | 当前按 `0x200` 试跑 |
| 单个 I/Q 分量 | 2 字节 |
| 数据类型 | `int16` 有符号整数 |
| 字节序 | 大端序（Big-Endian） |
| 排列方式 | `I0,Q0,I1,Q1,I2,Q2...` |
| 单个 IQ 对 | 4 字节 |

若按当前口径解释完整文件：

```text
(256,000,512 - 512) / 4 = 64,000,000 组 IQ
64,000,000 / 12,800,000 = 5 秒
```

这个结果与“12.8 MHz 采样率下录制 5 秒”的场景高度一致。

## 6. 观察结果

从 `0x200` 起按大端 `int16` 解码时，可看到：

- 前几组 IQ 样本为连续小整数
- 数据形态与弱信号背景下的 IQ 更匹配
- 与 `0x263` 这类错位起点相比明显更自洽

同时，`0x200` 之前表现为：

- 少量头字段
- 大量零填充
- 符合“参数块 + 保留区 + IQ 数据区”的典型结构

## 7. 待继续确认的内容

当前仍未彻底坐实的部分：

- `0x200` 是否为最终固定头长
- `0x0008` 与 `0x0020 - 0x00FF` 区域的具体含义
- `0x0110 / 0x0114` 的准确业务语义
- 是否存在更多设备型号下不同头结构

后续最好通过更多**已知录制时长**的 CAP 文件继续交叉验证。

## 8. 当前可落地结论

本轮实现层面可安全采用：

1. 输入文件使用 `.cap`
2. 版本号从 `0x0000` 读取
3. 带宽从 `0x0010` 读取
4. 中心频率从 `0x0018` 读取
5. 实际采样率按 `bandwidth_hz * 1.28` 推得
6. IQ 数据按 `0x200` 起始、大端 `int16` 交织试跑

本记录用于给当前联调、页面预览和算法接入提供一致事实基础。
