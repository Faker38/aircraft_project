# CAP 格式验证记录

## 1. 目的

本记录用于整理当前民航 `CAP` 样本文件的已验证结构信息，为“信号预处理”页中的导入预览和后续正式解析器开发提供依据。

本阶段目标仅为格式验证与只读预览，不将该数据接入无人机训练、识别或数据集管理主流程。

## 2. 分析样本

当前工作区样本如下：

- `IQ_2025_01_09_13_55_30.cap`，大小 `256,000,512` 字节
- `head.cap`，大小 `1,048,576` 字节

已验证：

- `head.cap` 与完整文件前 `1 MB` 前缀完全一致
- `head.cap` 可作为头部与早期 IQ 数据的快速探针样本

## 3. 已确认字段

### 3.1 文件结构总览

当前样本可按下列结构解释：

```text
┌─────────────────────────────────────────┐
│ 文件头（固定 11264 字节 = 0x2C00）      │
├─────────────────────────────────────────┤
│ IQ 数据区（从 0x2C00 到文件末尾）       │
│ 大端 int16，I/Q 交织：I0,Q0,I1,Q1...    │
└─────────────────────────────────────────┘
```

### 3.2 已确认头部字段

| 偏移 | 长度 | 类型 | 当前值 | 说明 |
| --- | --- | --- | --- | --- |
| `0x0000` | 8 字节 | ASCII 字符串（null 结尾） | `B.01.00` | 文件格式版本号 |
| `0x0010` | 8 字节 | `float64` 大端序 | `10000000.0` | 采样率，单位 Hz |
| `0x0018` | 8 字节 | `float64` 大端序 | `1090000000.0` | 中心频率，单位 Hz |
| `0x0110` | 4 字节 | `uint32` 大端序 | `62500` | 疑似每帧采样数，当前已稳定读出 |
| `0x0114` | 4 字节 | `uint32` 大端序 | `1024` | 疑似块大小或 FFT 长度，当前已稳定读出 |

### 3.3 IQ 数据格式

| 项目 | 当前结论 |
| --- | --- |
| IQ 起始偏移 | `0x2C00` |
| 单个 I/Q 分量 | 2 字节 |
| 数据类型 | `int16` 有符号整数 |
| 字节序 | 大端序（Big-Endian） |
| 排列方式 | `I0,Q0,I1,Q1,I2,Q2...` |
| 单个 IQ 对 | 4 字节 |
| IQ 对总数 | `(文件大小 - 11264) / 4` |

当前完整样本对应：

```text
(256,000,512 - 11,264) / 4 = 63,997,312 组 IQ
```

## 4. 当前观测结果

### 4.1 前几组 IQ 样本

从 `0x2C00` 起按大端 `int16` 解码，前几组 IQ 样本如下：

```text
(-7, 7), (-3, 3), (3, -1), (-1, -1), (0, 4), (-5, 1)
```

这一结果与弱信号/背景噪声场景下的 IQ 分布更吻合，也明显优于小端解释时出现的异常大幅值。

### 4.2 统计特征

基于前 `1 MB` IQ 数据窗口的统计结果可作为当前预览基准：

- I、Q 均值接近 `0`
- I、Q 标准差量级接近
- 数值范围较小，符合弱信号背景下的 ADS-B 采样特征

这说明：

1. 当前字节序判断与数据起始偏移判断是自洽的
2. `head.cap` 作为头部探针样本具备代表性

## 5. 待确认字段

以下内容尚未完成可靠验证，当前不进入正式业务解释：

- `0x0008` 的 8 字节参数
- `0x0020 - 0x00FF` 的其他设备参数
- `0x0104 / 0x0108 / 0x010C` 等标志位
- 带宽、增益、时间戳编码方式
- 是否存在更细粒度的帧级目录或保留字段含义

这些字段需要更多不同采样率、不同中心频率、不同带宽条件下的 `.cap` 文件交叉比对。

## 6. 当前保守结论

当前可用于实现的结论如下：

1. 该 `CAP` 文件不是标准通用容器，而是设备私有二进制格式
2. 文件头长度固定按 `0x2C00` 处理
3. 已确认字段可安全用于只读导入预览
4. IQ 数据区可按大端 `int16` 交织格式读取
5. 当前这批文件属于民航 ADS-B 数据，仅用于格式验证与导入链路验证

## 7. 后续建议

下一阶段建议保持轻量推进：

1. 继续扩充 `CapProbe`，只增加已验证字段，不提前解释未确认偏移
2. 在“信号预处理”页保留导入预览能力，不直接进入真实切片服务
3. 后续拿到更多不同参数的 `.cap` 文件后，再做头部字段交叉验证
4. 等字段稳定后，再开发正式解析器、样本切片和持久化逻辑

## 8. 本阶段边界

- 不把该民航数据纳入无人机识别训练
- 不把该民航数据混入现有数据集管理样本
- 不做数据库落库
- 不做真实预处理切片实现

本记录用于为当前 CAP 导入预览和后续解析器开发提供一致的事实基础。
