# 软件架构与流程说明

## 1. 软件定位与当前阶段

本软件用于无人机射频信号识别项目的桌面端联调与流程验证，围绕以下主流程展开：

1. 数据采集
2. 信号预处理
3. 数据集管理
4. 模型训练
5. 无人机识别

当前版本仍属于 Qt 联调原型阶段，重点解决页面结构、流程清晰度、模块边界和联调入口。数据集管理已接入 SQLite 本地数据库，预处理模块已接入外部 CAP 版算法；设备真实控制、真实训练调度和真实识别推理仍待接入。

## 2. 软件总体结构

### 2.1 UI 层

- 主窗口 `MainWindow`
- 6 个主页面：
  - 工程总览
  - 数据采集
  - 信号预处理
  - 数据集管理
  - 模型训练
  - 无人机识别
- 统一主题、卡片组件、状态标签和顺滑滚动

### 2.2 逻辑层

- 页面内部 mock 数据与联调逻辑
- 数据采集页的设备连接与采集记录 mock
- 预处理页的 CAP 探针、参数配置与外部算法联调
- 数据集页的映射维护、自动标注、人工复核、SQLite 持久化与版本生成
- 训练页可读取数据集版本详情和 manifest，识别页可消费已整理样本

### 2.3 数据层

当前数据集管理已接入 SQLite 本地数据库，数据库文件位于 `db/aircraft_project.sqlite3`。现阶段已落库对象如下：

| 数据对象 | 作用 |
| --- | --- |
| `raw_files` | 保存 `.cap` 原始文件路径与采样参数 |
| `preprocess_tasks` | 保存预处理任务参数、候选段数、有效段数和输出样本数 |
| `samples` | 保存预处理输出样本路径、标签、复核状态和是否纳入数据集 |
| `dataset_versions` | 保存数据集版本、任务类型、样本数、划分策略和标签分布 |
| `dataset_items` | 保存数据集版本与样本的关联关系、标签和划分集合 |

模型训练任务、模型文件、导出记录和正式评估结果后续仍建议单独扩展模型库。

## 3. 主导航结构与页面职责

| 页面 | 作用 |
| --- | --- |
| 工程总览 | 汇总流程状态、设备状态和关键指标 |
| 数据采集 | 连接 3943B，配置参数并生成原始 `.cap` 文件记录 |
| 信号预处理 | 选择 `.cap` 文件，读取头信息、执行预处理并生成样本记录 |
| 数据集管理 | 对预处理输出样本做标签维护、人工复核、数据库管理与数据集生成 |
| 模型训练 | 选择数据集版本，读取 manifest，执行训练联调、查看结果并导出模型 |
| 无人机识别 | 加载模型，对样本执行类型识别与个体识别联调 |

### 3.1 工程总览

- 查看当前流程阶段、设备状态和关键指标
- 提供主流程入口
- 不承载具体业务操作

### 3.2 数据采集

- 配置设备 IP、端口和 VISA 地址
- 模拟设备连接、断开和 `*IDN?` 查询
- 设置采集参数并生成原始文件记录
- 展示采集日志和文件列表

### 3.3 信号预处理

- 选择 `.cap` 原始文件
- 读取 CAP 头信息并执行导入预览
- 展示分析带宽、实际采样率、中心频率、IQ 统计摘要和前若干组 IQ 样本
- 配置切片长度、阈值、最小带宽、最小时长、模型路径和输出目录
- 后台执行外部 CAP 版预处理算法
- 展示处理日志、信号段结果表和样本输出记录

### 3.4 数据集管理

- 接收预处理输出样本
- 维护设备编号与标签映射关系
- 按设备和状态筛选样本
- 支持自动标注、单条人工复核、纳入/排除控制
- 支持样本和数据集版本的 SQLite 持久化
- 支持删除选中样本、删除选中版本和一键清空样本数据库
- 设置训练、验证、测试集划分比例
- 配置划分策略并生成数据集版本和 `manifest.json`

### 3.5 模型训练

- 选择任务类型和数据集版本
- 读取数据库中的数据集版本详情和 manifest 路径
- 检查样本文件是否存在、标签是否完整、训练/验证/测试划分数量
- 配置训练参数
- 展示训练日志、混淆矩阵占位区和评估结果
- 在训练页内部完成模型导出配置

### 3.6 无人机识别

- 加载类型识别模型和个体识别模型
- 选择待识别样本
- 执行单样本识别
- 展示预测结果、置信度和命中状态

## 4. 端到端业务流程

当前软件的标准流程如下：

1. 在“数据采集”页面连接设备并生成原始记录文件
2. 在“信号预处理”页面读取 CAP 头信息，执行信号筛选与样本输出
3. 在“数据集管理”页面完成映射维护、自动标注、人工复核、数据库保存和数据集划分
4. 在“模型训练”页面读取数据集版本详情和 manifest，完成数据检查、训练联调、评估展示和导出
5. 在“无人机识别”页面加载模型并执行类型或个体识别验证

补充说明：

- 当前不再保留公开数据集导入作为正式用户路径
- 后续统一以你们自己的采集与预处理输出样本作为测试对象
- 当前 CAP 联调口径按 `0x200 / 512` 字节头长试跑，带宽与实际采样率分开处理

## 6. 当前原型边界

当前界面已具备：

- 主流程导航与页面分工
- 统一视觉风格与顺滑滚动
- 数据集管理页的 SQLite 标注与版本生成闭环
- CAP 文件导入预览与外部算法联调
- 数据集版本 manifest 生成与训练页数据检查
- 统一的表单、表格、状态与日志界面
- CAP 版预处理算法对接说明

当前界面尚未接入：

- 真实 3943B 控制协议
- 真实模型训练调度
- 真实推理与识别服务

说明：

- 当前 CAP 头结构按 `0x200 / 512` 字节试跑，不作为永久铁结论
- 当前系统已具备外部预处理算法联调入口，但仍需继续验证头结构与检出效果
- 当前民航 CAP 文件仅用于格式验证，不进入无人机识别链路

## 8. 两分钟演示路径

建议演示时按以下顺序说明：

1. 打开软件，进入“信号预处理”，展示 CAP 文件选择、头信息预览和预处理结果区。
2. 进入“数据集管理”，展示样本列表、手动填写标签、映射表自动标注和“纳入数据集”勾选。
3. 生成数据集版本，说明系统会写入 SQLite，并生成 `data/datasets/{version_id}/manifest.json`。
4. 进入“模型训练”，选择刚生成的数据集版本，点击“数据检查”，展示 manifest 路径、标签分布和样本文件检查结果。
5. 简要展示删除样本、删除版本和清空样本数据库按钮，但不实际执行清空操作。
