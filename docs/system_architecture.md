# 软件架构与流程说明

## 1. 软件定位与当前阶段

本软件用于无人机射频信号识别项目的桌面端联调与流程验证，围绕以下主流程展开：

1. 数据采集
2. 信号预处理
3. 数据集管理
4. 模型训练
5. 无人机识别

当前版本仍属于 Qt 联调原型阶段，重点解决页面结构、流程清晰度、模块边界和联调入口。数据集管理已接入 SQLite 本地数据库，预处理模块已接入外部 CAP 版算法；类型识别已接入本地真实训练与真实推理，数据采集页已补齐 USRP 真实采集 V1，个体识别真实链路仍待接入。

## 2. 软件总体结构

### 2.1 UI 层

- 主窗口 `MainWindow`
- 6 个主页面：
  - 工程总览
  - 数据采集
  - 信号预处理
  - 数据集管理
  - 模型训练
  - 无人机识别
- 统一主题、卡片组件、状态标签和顺滑滚动

### 2.2 逻辑层

- 页面内部 mock 数据与联调逻辑
- 数据采集页的 3943B 联调模拟入口与 USRP 真实采集 V1
- 预处理页的 CAP 探针、参数配置与外部算法联调
- 数据集页的映射维护、自动标注、人工复核、SQLite 持久化与版本生成
- 数据集版本生成已实现真实划分：按样本随机分层固定随机种子，按设备个体隔离会阻止设备数不足的版本
- 训练页可读取数据集版本详情和 manifest，并执行本地真实类型识别训练
- 训练页内置外部标注测试集批量评估能力
- 识别页可消费已整理样本和已训练模型记录，执行真实类型识别推理，并提示样本是否超出模型训练适用域

### 2.3 数据层

当前数据集管理已接入 SQLite 本地数据库，数据库文件位于 `db/aircraft_project.sqlite3`。现阶段已落库对象如下：

| 数据对象 | 作用 |
| --- | --- |
| `raw_files` | 保存 `.cap` 原始文件路径与采样参数 |
| `preprocess_tasks` | 保存 CAP 与 USRP 演示预处理任务参数、候选段数、有效段数和输出样本数 |
| `samples` | 保存预处理输出样本路径、标签、复核状态和是否纳入数据集 |
| `dataset_versions` | 保存数据集版本、任务类型、样本数、划分策略和标签分布 |
| `dataset_items` | 保存数据集版本与样本的关联关系、标签和划分集合 |
| `trained_models` | 保存类型识别训练产物、模型路径、标签空间和评估指标 |
| `label_mappings` | 保存设备编号到类型/个体标签的自动标注映射 |

当前类型识别模型产物会落盘到 `data/models/`，并通过 `trained_models` 表供训练页和识别页统一消费。后续如需扩展个体识别和更复杂的导出物，可继续细化模型库。

## 3. 主导航结构与页面职责

| 页面 | 作用 |
| --- | --- |
| 工程总览 | 汇总流程状态、设备状态和关键指标 |
| 数据采集 | 在 3943B 联调模拟或 USRP 真实采集 V1 下配置参数并生成原始文件记录 |
| 信号预处理 | 选择 `.cap` 文件，读取头信息、执行预处理并生成样本记录 |
| 数据集管理 | 对预处理输出样本做标签维护、人工复核、数据库管理与数据集生成 |
| 模型训练 | 选择数据集版本，读取 manifest，执行真实类型识别训练、查看结果并管理模型产物 |
| 无人机识别 | 加载真实类型识别模型，对样本执行类型识别；个体指纹识别入口保留，真实服务待接入 |

### 3.1 工程总览

- 查看当前流程阶段、设备状态和关键指标
- 提供主流程入口
- 不承载具体业务操作

### 3.2 数据采集

- 支持 `3943B 联调模拟` 与 `USRP 真实采集`
- 3943B 模式下保留设备连接、断开和 `*IDN?` 查询入口，当前为协议待接入的联调模拟
- USRP 模式下通过外部采集命令执行真实采集
- USRP 第一版输出独立 `.iq/.bin` 文件与同目录元数据 `.json`
- 展示采集日志和文件列表

### 3.3 信号预处理

- 选择 `.cap` 原始文件
- 读取 CAP 头信息并执行导入预览
- 展示分析带宽、实际采样率、中心频率、IQ 统计摘要和前若干组 IQ 样本
- 配置切片长度、阈值、最小带宽、最小时长、模型路径和输出目录
- 后台执行外部 CAP 版预处理算法
- 可切换到 `USRP IQ 演示预处理`，把 B210 `.iq + .json` 切片成 `.npy` 样本并自动建议频点标签
- CAP 与 USRP 预处理任务都会写入 `preprocess_tasks`，便于删除关联、追溯参数和复盘演示流程
- 展示处理日志、信号段结果表和样本输出记录

### 3.4 数据集管理

- 接收预处理输出样本
- 维护设备编号与标签映射关系
- 按设备和状态筛选样本
- 支持自动标注、单条人工复核、纳入/排除控制
- 支持样本和数据集版本的 SQLite 持久化
- 支持删除选中样本、删除选中版本和一键清空样本数据库
- 设置训练、验证、测试集划分比例
- 配置真实划分策略并生成数据集版本和 `manifest.json`：`按样本随机分层` 会在标签内随机分层，`按设备个体隔离` 会保证同一设备不跨集合
- 提供孤立本地文件预览清理入口，只扫描项目 `data/raw`、`data/samples`、`data/datasets`、`data/models`
- 当前 demo 建议优先使用 `类别A / 类别B / 类别C` 这类中性标签组织样本批次

### 3.5 模型训练

- 选择任务类型和数据集版本
- 读取数据库中的数据集版本详情和 manifest 路径
- 检查样本文件是否存在、标签是否完整、训练/验证/测试划分数量
- 对 `.npy` IQ 样本提取轻量特征，并使用 `RandomForest` 执行本地真实训练
- 展示训练日志、测试集混淆矩阵、类别指标和模型文件路径
- 生成 `model.joblib` 与 `metadata.json`，并写入 `trained_models` 模型记录
- 在训练页内导入外部带标签 CSV，对真实模型执行批量测试并输出 `data/evaluations/` 报告

### 3.6 无人机识别

- 加载类型识别真实模型；个体指纹识别入口保留，真实模型待接入
- 选择待识别样本
- 类型识别页执行单样本真实推理
- 展示预测结果、预测置信度、适用域提示、命中状态和模型标签空间
- 预测置信度表示当前闭集模型的最大类别概率，不等同于测试准确率；域外样本仍可推理，但结果仅供参考
- 个体指纹识别页当前保留功能入口，真实推理服务待接入

## 4. 端到端业务流程

当前软件的标准流程如下：

1. 在“数据采集”页面完成 3943B 联调模拟或 USRP 真实采集，并生成原始记录文件
2. 在“信号预处理”页面读取 CAP 头信息并执行 CAP 算法预处理，或读取 USRP IQ 元数据并执行演示预处理
3. 在“数据集管理”页面完成映射维护、自动标注、人工复核、数据库保存和数据集划分
4. 在“模型训练”页面读取数据集版本详情和 manifest，完成数据检查、真实类型识别训练和模型落盘
5. 在“无人机识别”页面加载训练完成的类型识别模型并执行真实单样本推理

补充说明：

- 当前不再保留公开数据集导入作为正式用户路径
- 后续统一以你们自己的采集与预处理输出样本作为测试对象
- 当前 CAP 联调口径按 `0x200 / 512` 字节头长试跑，带宽与实际采样率分开处理
- 当前民航 CAP 仅用于流程演示与格式验证，demo 标签建议使用中性类别名

## 6. 当前原型边界

当前界面已具备：

- 主流程导航与页面分工
- 统一视觉风格与顺滑滚动
- 数据集管理页的 SQLite 标注与版本生成闭环
- CAP 文件导入预览与外部算法联调
- 数据集版本 manifest 生成与训练页真实训练
- 类型识别真实模型产物落盘与识别页真实推理
- 训练页内外部测试集批量评估
- 统一的表单、表格、状态与日志界面
- CAP 版预处理算法对接说明

当前界面尚未接入：

- 真实 3943B 控制协议
- 个体识别真实训练与真实推理服务
- 云上 GPU 训练通道

说明：

- 当前 CAP 头结构按 `0x200 / 512` 字节试跑，不作为永久铁结论
- 当前系统已具备外部预处理算法联调入口，但仍需继续验证头结构与检出效果
- 当前民航 CAP 文件仅用于格式验证，不进入无人机识别链路

## 8. 两分钟演示路径

建议演示时按以下顺序说明：

1. 打开软件，进入“信号预处理”，展示工作区根目录下自动扫描到的 CAP 文件、头信息预览和预处理结果区。
2. 进入“数据集管理”，展示样本列表、手动填写标签、映射表自动标注和“纳入数据集”勾选。
3. 生成数据集版本，说明系统会写入 SQLite，并生成 `data/datasets/{version_id}/manifest.json`。
4. 进入“模型训练”，选择刚生成的数据集版本，展示数据检查结果并执行一次真实类型识别训练。
5. 进入“无人机识别”，加载刚训练出的模型，对一条样本执行真实类型识别。
6. 简要展示删除样本、删除版本和清空样本数据库按钮，但不实际执行清空操作。
