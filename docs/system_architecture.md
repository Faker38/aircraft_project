# 软件架构与流程说明

## 1. 软件定位与当前阶段

本软件用于无人机射频信号识别项目的桌面端联调与流程验证，覆盖以下业务链路：

1. 设备接入与原始数据采集
2. 信号预处理与样本生成
3. 数据集管理与标注维护
4. 模型训练、评估与导出
5. 无人机识别验证

当前版本仍属于 Qt 界面原型阶段，重点解决页面结构、操作流程、模块边界与导入预览能力。设备通信、样本切片、训练调度、识别推理和数据库持久化等能力尚未接入真实业务实现。

## 2. 软件总体结构

软件当前按三层理解最合适：

### 2.1 UI 层

- 主窗口 `MainWindow`
- 6 个主页面：
  - 工程总览
  - 数据采集
  - 信号预处理
  - 数据集管理
  - 模型训练
  - 无人机识别
- 统一主题、卡片组件、顺滑滚动和状态标签

### 2.2 逻辑层

- 页面内部 mock 数据与交互逻辑
- 数据集页的映射维护、自动标注、人工复核
- 预处理页的 CAP 只读探针与导入预览

### 2.3 数据层

- 当前仅保留目录约定与文件流转位置
- 尚未接入数据库、真实样本切片、真实训练和推理服务

### 2.4 当前文件流

当前软件的目标文件流如下：

```text
采集原始 .cap
    -> 信号预处理 / CAP 导入预览
    -> 数据集管理（映射、标注、划分）
    -> 模型训练 / 导出
    -> 无人机识别验证
```

## 3. 主导航结构与页面职责

当前主导航共 6 个页面：

| 页面 | 作用 |
| --- | --- |
| 工程总览 | 汇总系统状态、流程入口和关键指标 |
| 数据采集 | 连接 3943B，配置参数并生成原始 `.cap` 文件 |
| 信号预处理 | 选择原始文件，执行 CAP 导入预览，并保留预处理参数配置 |
| 数据集管理 | 管理样本标注、编号映射和数据集版本 |
| 模型训练 | 配置训练任务、查看评估结果并导出模型 |
| 无人机识别 | 使用已训练模型执行类型识别与个体指纹识别 |

### 3.1 工程总览

- 查看当前流程阶段、设备状态和关键指标
- 提供主流程入口
- 不承载具体业务操作

### 3.2 数据采集

- 配置设备 IP、端口和 VISA 地址
- 模拟设备连接、断开和 `*IDN?` 查询
- 设置采集参数并生成原始文件记录
- 展示采集日志和文件列表

输入：

- 设备连接参数
- 采集参数

输出：

- 原始 `.cap` 文件记录
- 采集状态与日志信息

### 3.3 信号预处理

- 选择 `.cap` 原始文件
- 读取 CAP 头信息并执行只读导入预览
- 展示已验证字段、IQ 统计摘要和前若干组 IQ 样本
- 保留切片长度、阈值、噪声基底等预处理参数界面
- 当前未接入真实切片服务

输入：

- 原始 `.cap` 文件
- 预处理参数

输出：

- CAP 头部预览信息
- IQ 数据摘要
- 后续真实样本切片的参数入口

### 3.4 数据集管理

页面内部包含两个页签：

#### 样本标注

- 维护设备编号与标签映射关系
- 按设备和标注状态筛选样本
- 支持自动标注与单条人工复核

#### 数据集构建

- 设置训练、验证、测试集划分比例
- 配置划分策略
- 查看各类别或个体的分集结果
- 管理历史数据集版本

输入：

- 样本文件与样本元数据
- 标注信息
- 数据集划分参数

输出：

- 已标注样本
- 数据集版本记录

### 3.5 模型训练

- 选择任务类型和数据集版本
- 配置训练参数
- 展示训练日志、混淆矩阵占位区和评估结果
- 在训练页内部完成模型导出配置

输入：

- 已构建数据集
- 训练参数
- 导出配置

输出：

- 训练模型
- 评估结果
- 导出交付文件

### 3.6 无人机识别

页面内部包含两个页签：

#### 无人机类型识别

- 加载类型识别模型
- 选择样本来源
- 执行单样本识别
- 展示类别置信度结果

#### 无人机个体指纹识别

- 加载个体识别模型
- 选择样本来源
- 执行个体指纹识别
- 展示个体候选结果和置信度

输入：

- 已训练模型
- 待识别样本

输出：

- 类型识别结果
- 个体识别结果

## 4. 端到端业务流程

当前软件的标准流程如下：

1. 在“数据采集”页面连接设备并生成原始记录文件
2. 在“信号预处理”页面读取 CAP 头信息并确认原始文件结构
3. 在“数据集管理”页面完成映射维护、自动标注、人工复核和数据集划分
4. 在“模型训练”页面完成训练、评估和导出
5. 在“无人机识别”页面加载模型并执行类型或个体识别验证

## 6. 当前原型边界

当前界面已具备：

- 主流程导航与页面分工
- 统一视觉风格与顺滑滚动
- 数据集管理页的可演示闭环
- CAP 文件只读导入预览
- 统一的表单、表格、状态与日志界面

当前界面尚未接入：

- 真实 3943B 控制协议
- 真实 `.cap` 样本切片与预处理服务
- 实际数据库持久化逻辑
- 真实模型训练调度
- 真实推理与识别服务

说明：

- 当前 CAP 导入预览仅用于确认头部结构与 IQ 数据布局
- 这不代表系统已经具备正式解析、切片或样本入库能力
- 当前民航 CAP 文件仅用于格式验证，不进入无人机识别链路
